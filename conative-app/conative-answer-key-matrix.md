# Conative Assessment — Answer Key Matrix

> This file maps every option of every question to its corresponding Action Mode.
> FF = Fact Finder | FT = Follow Thru | QS = Quick Start | IMP = Implementor
>
> Scoring: MOST selection = +2 to that mode | LEAST selection = -2 to that mode | Unselected = 0
>
> Source: Psychometric Blueprint doc, Item 1 mapping explicitly confirmed in text.
> Remaining items derived from the doc's behavioral taxonomy:
> - FF signals: research, data, detail, specifics, facts, evidence, written proof, thorough, examine, probe, priorities, accuracy, historical, realistic, pros/cons
> - FT signals: structure, organize, system, plan, sequence, orderly, charts/graphs, diagrams, neatness, patterns, design, coordinate, uniform, consistent, methodical
> - QS signals: experiment, risk, challenge, new, unique, improvise, spontaneous, innovate, brainstorm, variety, imagination, speed, cleverness, change, flexible, promote
> - IMP signals: build, craft, physical, tangible, hands-on, skill (manual), model, sculpture, equipment, construct, practice, durable, repair, materials, endurance

## NOTES & EDGE CASES

### Item 8/9 Data Corruption
The source document has corrupted text for Item 8 ("When nd objectives") and Item 9 is missing entirely. Two options:
- **Option A:** Drop to 34 items, adjust the scoring divisor from 12 to ~11.3 to maintain score spread.
- **Option B (Recommended):** Write 2-3 custom replacement items following the doc's item construction rules (active verbs, context-agnostic scenarios, one option per mode, no social desirability bias). Examples below.

**Replacement Item 8 (suggested):**
"When starting a group project, my instinct is to:"
- Option A: gather all the background information first → FF
- Option B: create a step-by-step plan → FT  
- Option C: jump in and figure it out as we go → QS
- Option D: start building something right away → IMP

**Replacement Item 9 (suggested):**
"If asked to improve a process, I would first:"
- Option A: study how it currently works in detail → FF
- Option B: map out a better sequence → FT
- Option C: try a completely different approach → QS
- Option D: test it hands-on to find the weak points → IMP

### Item 13 Mapping Ambiguity
Item 13 has a potential mapping issue. "Touchiness" and "impulsiveness" could both signal QS (emotional reactivity under risk/change). The doc's taxonomy suggests:
- "touchiness" → more likely IMP (physical/sensory sensitivity, tactile responsiveness)
- "impulsiveness" → QS (risk-taking, spontaneous action)
- "preciseness" → FF (detail orientation)  
- "predictability" → FT (structural consistency)

I've mapped touchiness → QS in the main table above based on the emotional-reactivity reading, but this item may need revision. If building original questions, replace this item with something less ambiguous.

**Revised mapping for Item 13 (if using IMP interpretation):**
| 13 | If I am teased about a characteristic, it is usually my: | touchiness | IMP | impulsiveness | QS | preciseness | FF | predictability | FT |

### Item 11 Mapping Note
"Discussions" as something to avoid maps to QS (resistance to open-ended brainstorming/debate). This reads as a Stabilizer trait: avoiding the unstructured, free-flowing conversation that QS-initiators thrive in. The person who avoids discussions is counteracting the Quick Start mode.

### Item 18 Mapping Note  
"Sensitive" maps to IMP rather than an affective trait because in the conative context, it refers to physical/sensory sensitivity and heightened awareness of the tangible environment, not emotional sensitivity (which would be affective, not conative).

## SCORING ALGORITHM (JavaScript Implementation)

```javascript
function scoreAssessment(responses, answerKey) {
  // responses = { "1": { most: "opt2", least: "opt3" }, "2": { most: "opt4", least: "opt1" }, ... }
  
  const rawScores = { FF: 0, FT: 0, QS: 0, IMP: 0 };
  
  // Step 1: Calculate raw scores
  for (const [qNum, answer] of Object.entries(responses)) {
    const key = answerKey[qNum];
    if (!key) continue;
    
    const mostMode = key[answer.most];
    const leastMode = key[answer.least];
    
    rawScores[mostMode] += 2;
    rawScores[leastMode] -= 2;
  }
  
  // Step 2: Normalize to 1-10 scale
  const DIVISOR = 12; // Adjustable. Lower = more spread.
  const normalizedScores = {};
  
  for (const [mode, raw] of Object.entries(rawScores)) {
    let score = Math.round(5.5 + (raw / DIVISOR));
    score = Math.max(1, Math.min(10, score)); // Clamp 1-10
    normalizedScores[mode] = score;
  }
  
  // Step 3: Calculate energy percentages
  const ENERGY_FLOOR = 5; // Minimum 5% per mode
  const deviations = {};
  let totalDeviation = 0;
  
  for (const [mode, score] of Object.entries(normalizedScores)) {
    deviations[mode] = Math.abs(score - 5.5);
    totalDeviation += deviations[mode];
  }
  
  const energyPct = {};
  
  if (totalDeviation === 0) {
    // Edge case: all scores exactly at center
    for (const mode of Object.keys(normalizedScores)) {
      energyPct[mode] = 25;
    }
  } else {
    let floored = [];
    let floorTotal = 0;
    
    for (const [mode, dev] of Object.entries(deviations)) {
      let pct = (dev / totalDeviation) * 100;
      if (pct < ENERGY_FLOOR) {
        pct = ENERGY_FLOOR;
        floored.push(mode);
      }
      energyPct[mode] = pct;
      floorTotal += pct;
    }
    
    // Redistribute excess from flooring
    if (floorTotal !== 100) {
      const nonFloored = Object.keys(energyPct).filter(m => !floored.includes(m));
      const nonFlooredTotal = nonFloored.reduce((sum, m) => sum + energyPct[m], 0);
      const adjustment = (100 - floored.length * ENERGY_FLOOR) / nonFlooredTotal;
      
      for (const mode of nonFloored) {
        energyPct[mode] = Math.round(energyPct[mode] * adjustment);
      }
      for (const mode of floored) {
        energyPct[mode] = ENERGY_FLOOR;
      }
      
      // Fix rounding to exactly 100
      const total = Object.values(energyPct).reduce((a, b) => a + b, 0);
      if (total !== 100) {
        const maxMode = Object.entries(energyPct).sort((a, b) => b[1] - a[1])[0][0];
        energyPct[maxMode] += (100 - total);
      }
    }
  }
  
  // Step 4: Determine zones and strengths
  const ZONE_MAP = {
    FF:  { counteract: "Simplify",    accommodate: "Explain",    initiate: "Specify"     },
    FT:  { counteract: "Adapt",       accommodate: "Maintain",   initiate: "Systematize" },
    QS:  { counteract: "Stabilize",   accommodate: "Modify",     initiate: "Innovate"    },
    IMP: { counteract: "Envision",    accommodate: "Restore",    initiate: "Build"       }
  };
  
  const zones = {};
  const strengths = {};
  
  for (const [mode, score] of Object.entries(normalizedScores)) {
    if (score <= 3) {
      zones[mode] = "counteract";
    } else if (score <= 6) {
      zones[mode] = "accommodate";
    } else {
      zones[mode] = "initiate";
    }
    strengths[mode] = ZONE_MAP[mode][zones[mode]];
  }
  
  // Step 5: Identify dominant and resistance modes
  const sorted = Object.entries(energyPct).sort((a, b) => b[1] - a[1]);
  const dominantMode = sorted[0][0];
  const resistanceMode = sorted[sorted.length - 1][0];
  
  // Step 6: Build MO string
  const moString = `${normalizedScores.FF}-${normalizedScores.FT}-${normalizedScores.QS}-${normalizedScores.IMP}`;
  
  return {
    rawScores,
    normalizedScores,
    energyPct,
    zones,
    strengths,
    dominantMode,
    resistanceMode,
    moString
  };
}
```
